# Volta Landing Page — Blank-Slate Design Direction

## Primary Reference

The design reference is specifically **https://datacurve.ai/products**.

This is not a product-catalog copy. Volta is still a landing page. The borrowed elements are the visual system: black editorial type, cream surfaces, sparse navigation, generous whitespace, and large typographic product cards.

---

## Visual Translation for Volta

**What to borrow from DataCurve products:**
- Cream canvas with near-black typography
- Large display serif headlines
- Minimal sans-serif body copy
- Product-page-style cards with hairline borders
- Sparse CTAs and restrained navigation
- High-contrast dark sections used sparingly
- Very little color beyond the neutral system

**What not to bring back:**
- Terracotta/sage accent palette
- Decorative gradients
- Hero mockup/dashboard visuals
- Icon-heavy feature cards
- Metric strips with unsupported claims
- Generic SaaS three-column templates

## Typography

DataCurve uses Exposure for display and Suisse Intl for body. Volta approximates that with:

- **Display:** Instrument Serif via `next/font/google`
- **Body:** Inter via `next/font/google`

Rules:
- Hero display type should be very large: up to ~112px desktop
- Display line-height should be tight: ~0.9–0.96
- Body type should be compact: 15–18px
- Labels should be small mono-style uppercase metadata

---

## Layout

- Hero is text-first, no mockup visual
- The Volta Workflow is the main product-card section
- Workflow cards should be large 2×2 cards, not tiny feature tiles
- Outputs can be 6 smaller product cards
- Trust can be the main high-contrast black section
- Sponsor/expert audience blocks are two large cards

---

## Content Principles

- No unsupported numeric stats
- Fixed/milestone pricing is explicit
- Copy should stay direct and short, inspired by Panacea's clarity
- Volta's core promise: scientists keep doing science; Volta handles the filing lifecycle
